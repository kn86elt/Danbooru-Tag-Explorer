## [0.6.4] — 2026-05-16

### Added
- **AI候補数コントロール（ − ×N + ↺）**
  - チップ列末尾に候補数コントロールを追加
  - `−` / `+` で候補数を 1 件ずつ増減し LLM に再リクエスト（範囲 1〜10）
  - `↺` ボタンで同じ候補数のまま再翻訳（ブレインストーミング用途）
  - 候補数は `localStorage` に保存され、次回起動時も維持
- チップ末尾に元の日本語入力ワードを追加し、AI 翻訳前の検索に切り替えられる
- **LLM タブ: 候補グループビュー**
  - 各コンセプトの候補を折りたたみグループ形式で表示
  - 厳格タグ解決（完全一致 → alias → トークン一致の順で Danbooru タグに正規化）

### Changed
- `POST /api/ai-translate` が `count` パラメータを受け付けるようになり、システムプロンプトの候補数指示を動的に変更
- LLM リクエストの `temperature` を 0.3 → 0.6 に変更（候補の多様性向上）
- `max_tokens` を候補数に応じてスケール（`150 + count × 30`）

### Fixed
- 暗転した候補チップをクリックした際に詳細モーダルが開いてしまう問題を修正（アクティブ切り替えのみに変更）

## [0.6.3] — 2026-05-16

### Added
- **AI検索: 複数候補チップ**
  - LLM が各コンセプトに対して複数の候補（デフォルト 3 件）を返すよう拡張（出力フォーマット: `日本語語句: candidate1 | candidate2 | candidate3`）
  - 候補の組み合わせからクエリバリアントを生成し、検索ボックス下にクリッカブルなチップとして表示（最大 16 件）
  - チップをクリックするとそのバリアントで即時再検索し、Enter →「一覧表示」にも反映

### Fixed
- AI 翻訳後の Enter →「一覧表示」に翻訳済みクエリを反映（`_lastAiQuery` で保持・リセット）
- LLM システムプロンプトにひらがな変換禁止指示を追加（`ja.csv` 逆引きのヒット率向上）

## [0.6.1] — 2026-05-15

### Added
- **[🤖] LLM 検索ボタン（検索ボックス右端）**
  - 結果件数に関わらず強制的に LLM 翻訳検索を発火
- **LLM連携タブ: コピー・クリアボタン追加**
- **LLM 出力の Danbooru タグ正規化**
  - LLM に `日本語語句: english keyword` のペア形式で出力させ、日本語側を `ja.csv` で逆引き
  - 逆引きで一致した場合は正規 Danbooru タグ名に置換（例: `頬杖` → `head_rest`）
  - 逆引き失敗時は英語キーワードをタグデータベースで直接解決（完全一致 → トークン一致）
  - `\_` → `_` のバックスラッシュエスケープ除去（`\(` `\)` は A1111 構文のため保持）
  - `replace-underscore` 設定に従いアンダースコア→スペース変換
  - 末尾カンマを自動付加

## [0.6.0] — 2026-05-15

### Added
- **スクラッチパッド: タブ切り替え**
  - 「プロンプト編集」「LLM連携(β)」の 2 タブを追加
  - LLM連携タブ: 日本語入力エリア・タグ出力エリア・右タグリスト

- **設定モーダル（⚙ ボタン）**
  - LLM バックエンド設定: Ollama / LM Studio / text-generation-webui / KoboldCpp / llama.cpp server / カスタム の 6 プリセット
  - プリセット切り替えで host / port / path / API キーを自動補完
  - モデル一覧取得（🔄）と選択セレクトボックス
  - モデルアンロードボタン（Ollama・LM Studio・text-gen-webui のみ）
  - 接続テストボタン
  - CSV パス設定（スタンドアロンモード。A1111 モードは読み取り専用表示）

- **LLM 変換（スクラッチパッド LLM連携タブ）**
  - 「→ 変換」ボタンで日本語テキストを `POST /api/ai-translate` に送信し Danbooru タグを生成
  - 変換中はボタンを「変換中...」表示・disabled に
  - エラー時はトースト通知

- **日本語検索の自動 LLM 翻訳**
  - 日本語クエリで検索結果ゼロの場合、最終キー入力から 800ms 後に LLM が自動発火
  - 翻訳中は「🤖 翻訳中...」ヒントを表示
  - 翻訳結果で再検索し、「🤖 AI翻訳」バッジを表示
  - LLM 未設定・接続不可はサイレントフォールバック

### Backend
- `POST /api/ai-translate` エンドポイントを追加（`server.py` / `scripts/danbooru_tag_explorer.py`）
- `GET /api/llm/models` にクエリパラメータ対応（保存前のフォーム値でテスト可能）
- `POST /api/llm/unload` エンドポイントを追加（Ollama・LM Studio・text-gen-webui 対応）
- `POST /api/settings` エンドポイントを追加（LLM 設定・CSV パスの保存）
- `_http_post` に `extra_headers` 引数を追加（API キー Authorization ヘッダー対応）

### Fixed
- モデルセレクタを `<input list="datalist">` から `<select>` に変更（datalist がフィルタリングにより 1 件しか表示されない問題を解消）
- 設定モーダルの接続テスト・モデル取得が `settings.json` の保存済み値ではなくフォームの現在値を使うよう修正

## [0.3.0] — 2026-05-11

### Added
- **グリッドビュー: カードサイズ変更スライダー**
  - グリッド表示時にツールバーへ S / M / L の 3 段階スライダーを追加
  - サイズはブラウザの localStorage に保存され、次回起動時も維持される
  - リスト表示中はスライダーを自動的に非表示にする

- **グリッドビュー: カードサイズ別のテキスト表示改善**
  - **S サイズ** (minmax 140px): フォントを 11px に縮小・カード高を詰める。アクションボタンをフロー外 (position: absolute) に退避させ `tag-name` が全幅を使用できるようにする。ホバー時にボタンを半透明 (opacity 0.75) で重ね表示
  - **M サイズ** (minmax 220px): 従来どおり (デフォルト)
  - **L サイズ** (minmax 320px): フォントを 13.5px に拡大。`-webkit-line-clamp: 2` による 2 行折り返し表示で長いタグ名も切れずに読める。日本語訳 (tag-ja) を合わせて表示

### Fixed
- **A1111 拡張: ブラウザキャッシュ問題の解消**
  - `build_index_html()` で `app.js` / `dte_app.css` の URL にファイル mtime をクエリパラメータとして付加するキャッシュバスティングを実装 (`?v=<mtime>`)
  - ファイル更新時に URL が自動で変わるため、A1111 再起動後にブラウザが古いキャッシュを使い続ける問題が解消される
  - `dte_static` ハンドラで JS / CSS レスポンスに `Cache-Control: no-cache, must-revalidate` ヘッダーを付加
